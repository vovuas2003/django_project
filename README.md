# Наумкин Владимир, С01-119. Django проект MathMaster для проверки устного счёта.

## Краткое описание

* Регистрация и авторизация пользователей
* Прохождение тестов из 10 сложений двузначных чисел с автоматической проверкой и подсчётом времени на стороне сервера
* Топ-10 пользователей по количеству тестов на 100% и по минимальному времени прохождения теста на 100%
* Просмотр обобщённой индивидуальной статистики и каждого пройденного теста
* Вывод графика прогресса времени выполнения тестов на 100% с построением прямой по МНК (интересное техническое решение)

## Возможности улучшения

* Новые тесты: трёхзначные числа, вычитание, умножение, целочисленное деление (и новые Топ-10 соответственно)
* Список всех пользователей (возможность зайти в чужую статистику)
* Статистика за выбранный период (не только за всё время)

## Запуск проекта (инструкция для Windows)

0. Установите Python (я запускал на 3.12.2)
1. Скачайте (склонируйте) репозиторий и зайдите в него из консоли (cmd)
2. python -m venv .venv
3. .venv\Scripts\activate
4. pip install -r requirements.txt
5. Создайте файл .env по подобию .env_example
6. python manage.py makemigrations
7. python manage.py migrate
8. python manage.py createsuperuser
9. python manage.py runserver
10. Перейдите в браузере по адресу http://127.0.0.1:8000/ (админка по адресу http://127.0.0.1:8000/admin/ но админ не является полноценным пользователем сайта - нет профиля)

## Запуск pylint

Кроме pylint установлен плагин pylint-django. Перед запуском pylint убедитесь, что виртуальное окружение активировано.

Команда pylint_run.bat запустит pylint с нужными опциями, сохранит результат в файл pylint_res.txt и выведет его содержимое на экран.

Текущая оценка pylint: 9.40

### Комментарии к выводу pylint

* Расширение стандартной модели пользователя можно реализовать более гибко без жёсткой привязки к User (например, вместо User использовать django.contrib.auth.get_user_model() или django.conf.settings.AUTH_USER_MODEL), этот вопрос требует дальнейшего изучения; но на данный момент единственная добавка joined_at, решил оставить как есть
* Тесты в данном проекте не затрагиваются даже по требованиям, я тестирую руками и этого как будто достаточно в данном случае
* Функция для визуализации прогресса с построением прямой МНК действительно самая сложная в данном проекте, решил не менять код в угоду pylint; а установить режим matplotlib без GUI обязательно надо, ведь без этого сервер падал, насколько я понял, именно из-за попытки отобразить окошко с графиком прогресса через Tkinter
